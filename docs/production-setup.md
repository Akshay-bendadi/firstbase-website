## Integration Steps

1. Install dependencies with `npm install` when cloning this project elsewhere.
2. Copy `.env.example` to `.env.local` and set the API base URL.
3. Start local development with `npm run dev`.
4. Deploy directly through Vercel.
5. Use `npm run build` locally when you want to verify the production bundle independently.
6. Run `npm audit --audit-level=high` manually before releases when needed.

## Quality Gate

- `npm run format` - write formatted files
- `npm run format:check` - verify formatting
- `npm run lint` - run Next.js linting
- `npm run build` - create the production build
- `npm run check` - run the full local quality gate
- `npm audit --audit-level=high` - optional manual dependency audit

Husky was not enabled for this scaffold. Deployment is handled directly by Vercel.

The scaffold writes `components.json`, `lib/utils`, and these local UI primitives: `button`, `card`, `badge`, `input`, `label`, `textarea`, `separator`. They are intentionally generated as source files so you can tune the design system without waiting on a registry call.

## Security Defaults

- Dependencies are installed with exact versions and `--save-exact`.
- Dependency audits are run manually when needed.
- Secret-bearing files such as `.env`, `.npmrc`, `*.pem`, `*.key`, and service account JSON files are ignored by git.
- `.env.example` uses fake placeholder values and must not contain real keys.
- The API client validates the public API base URL, sets a 15 second timeout, and blocks cross-origin absolute requests from that client instance.
- Next.js security headers, including a baseline Content Security Policy, are configured in `next.config`.
